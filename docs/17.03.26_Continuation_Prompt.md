## Next Step: Implement Auth (TOP PRIORITY)

This is the very next thing to build. Do not migrate more endpoints until auth is done.

### Plan

1. **Install dependencies**
   ```bash
   npm install jsonwebtoken bcrypt
   ```

2. **Build these endpoints in order:**
   - `POST /auth/register` — email + password, inserts into `users` table, returns JWT
   - `POST /auth/login` — verifies password hash, returns JWT
   - `GET /auth/me` — returns current user info from token

3. **Create `authMiddleware.js`** — a middleware function that:
   - Reads `Authorization: Bearer <token>` header
   - Verifies JWT
   - Attaches `req.user = { id, email, name }` to the request
   - Returns 401 if missing or invalid

4. **Wire `user_id` into all protected routes** — starting with:
   - `POST /metadata`
   - `PATCH /releases/:releaseId/metadata`
   - `GET /releases/` (filter by `user_id`)
   - `GET /releases/:releaseId` (verify ownership)
   - All future endpoints

### JWT Strategy
- Secret stored in `.env` as `JWT_SECRET`
- Token expiry: 7 days
- Payload: `{ userId, email, name }`

### Multi-Tenancy Rule
**Every single DB query that reads or writes user data MUST include `WHERE user_id = $X`.**
This is the core isolation guarantee. Never query without it on user-owned tables.

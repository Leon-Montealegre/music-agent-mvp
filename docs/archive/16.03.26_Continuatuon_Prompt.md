## Phase 2 — Cloud Migration (IN PROGRESS)

### Status: Planning complete. Build not yet started.

### Goal
Move the app from local JSON files to a hosted cloud stack accessible from anywhere, with multi-tenant support (each artist has their own login and isolated catalogue).

---

### ✅ All Architecture Decisions — Final

| Layer | Service | Notes |
|-------|---------|-------|
| Frontend | Vercel | Free tier, auto-deploys from GitHub on every push |
| Backend | Railway (Node.js/Express) | Free 30-day trial → free tier $1/mo credit → Hobby $5/mo |
| Database | Railway PostgreSQL | Included in Railway project, same plan |
| File Storage | Cloudflare R2 | Free up to 10 GB, then $0.015/GB/month. No egress fees. |
| Auth | NextAuth.js | Google login + email/password. User's choice. |

**File size limit:** 1 GB per file, uniform across all upload types (audio, video, artwork, documents).

**Multi-tenancy:** Multi-tenant from day one. Each artist has their own login and fully isolated catalogue. No SaaS layer yet (no public signup, no billing). Artists are invited via a direct signup link shared manually.

**IDs:** UUIDs internally for all database primary keys. The existing `releaseId` string format (`YYYY-MM-DD_ArtistName_Title`) is kept as a `slug` field for human-readable URLs. URLs stay identical to today.

**No data migration:** Existing JSON files are test/dummy data. Start fresh on PostgreSQL.

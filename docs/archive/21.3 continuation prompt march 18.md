## Remaining Bug: Artwork Not Showing on Homepage

### Root Cause
The `<img>` tag on the homepage makes a plain browser request to:
```
https://music-agent-mvp-production.up.railway.app/releases/{id}/artwork
```
This endpoint requires `Authorization: Bearer <token>` but browser `<img>` tags cannot send headers — so they always get a 401 and silently fail.

### Fix Required
Remove `authMiddleware` from the artwork GET endpoints in `server.js`.
Artwork images are not sensitive — anyone who knows the URL can view them anyway.

Find these endpoints in `file-handler/server.js`:
```
GET /releases/:releaseId/artwork
GET /collections/:collectionId/artwork
```

And remove `authMiddleware` from both. They currently look like:
```js
app.get('/releases/:releaseId/artwork', authMiddleware, async (req, res) => {
```

Change to:
```js
app.get('/releases/:releaseId/artwork', async (req, res) => {
```

Do the same for the collections artwork endpoint. Then push to `cloud-migration`.

Also check and fix the same issue for:
- `GET /releases/:releaseId/notes/files/:filename` (used in notes attachments)
- `GET /releases/:releaseId/files/:fileType/:filename` (used for audio/video serving in player)
- Collection equivalents of the above

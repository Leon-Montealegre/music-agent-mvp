## DEPLOYMENT GUIDE

### Step 1: Check Migration 003

Open Railway dashboard → your PostgreSQL database → Query tab. Run:

```sql
SELECT column_name FROM information_schema.columns WHERE table_name = 'distribution_entries' AND column_name = 'entry_notes';
```

If it returns nothing, paste the contents of `file-handler/migrations/003_add_entry_fields.sql` and run it.

### Step 2: Deploy Backend to Railway

1. Go to [railway.app](https://railway.app) → your project (the one with PostgreSQL)
2. Click **"+ New"** → **"GitHub Repo"** → select `Leon-Montealegre/music-agent-mvp`
3. Railway will auto-detect Node.js
4. In the service settings:
   - **Root directory**: `file-handler`
   - **Build command**: `npm install` (Railway does this by default)
   - **Start command**: `npm start` (reads from package.json `"start": "node server.js"`)
5. Go to the **Variables** tab and add these env vars:
   - `DATABASE_URL` — copy from your existing PostgreSQL service (Railway provides this as `${{Postgres.DATABASE_URL}}` reference variable)
   - `JWT_SECRET` — generate one: `openssl rand -base64 32`
   - `R2_ACCOUNT_ID` — from Cloudflare dashboard
   - `R2_ACCESS_KEY_ID` — from Cloudflare R2 API tokens
   - `R2_SECRET_ACCESS_KEY` — from Cloudflare R2 API tokens
   - `R2_BUCKET_NAME` — your R2 bucket name
   - Note: `PORT` is set automatically by Railway — do NOT add it manually
6. Go to **Settings** → **Networking** → **Generate Domain** to get your public URL
7. Test: `curl https://your-railway-url.up.railway.app/health`
   - Should return: `{ "status": "ok" }`

**Important**: Make sure Railway is deploying from the `cloud-migration` branch, not `main`. You can set this in Settings → Source → Branch.

### Step 3: Deploy Frontend to Vercel

1. Go to [vercel.com](https://vercel.com) → **"Add New Project"**
2. Import `Leon-Montealegre/music-agent-mvp` from GitHub
3. Configure:
   - **Root directory**: `frontend`
   - **Framework**: Next.js (auto-detected)
4. Add environment variables:
   - `NEXT_PUBLIC_API_URL` = `https://your-railway-url.up.railway.app` (the URL from Step 2.6, no trailing slash)
   - `NEXTAUTH_SECRET` = generate one: `openssl rand -base64 32`
   - `NEXTAUTH_URL` = `https://your-vercel-url.vercel.app` (you'll know this after first deploy — you can update it)
5. Click **Deploy**
6. After deploy, update `NEXTAUTH_URL` if the Vercel URL changed

**Important**: In Vercel settings, set the Git branch to `cloud-migration`.

### Step 4: End-to-End Verification

Once both are deployed:

1. **Health check**: `curl https://your-railway-url.up.railway.app/health`
2. **Open the Vercel URL** in your browser — you should see the login page
3. **Register a user**: Enter email + password on the login page
   - Or use the API: `curl -X POST https://your-railway-url/auth/register -H "Content-Type: application/json" -d '{"email":"you@example.com","password":"yourpassword"}'`
4. **Log in** — you should see the main dashboard
5. **Create a release** — title, artwork, audio should all work
6. **Test data isolation**: Register a second user, verify they can't see user 1's releases
7. **Test file serving**: Upload artwork → verify it displays (served from R2)

## Next Step: DELETE /releases/:releaseId (TOP PRIORITY)

This endpoint currently deletes from **local disk only**. It needs to delete from **PostgreSQL**.

### What to build:
Replace the current `DELETE /releases/:releaseId` handler (around line 598 in `server.js`)
with one that:
1. Is protected by `authMiddleware`
2. Looks up the release UUID from the `slug` (with `WHERE slug = $1 AND user_id = $2`)
3. Returns 404 if not found or doesn't belong to this user
4. Deletes the row from `releases` table (cascade deletes handle all related rows automatically)

The SQL cascade rules in the schema mean deleting a release automatically deletes:
- All its `distribution_entries`
- All its `files`
- All its `notes`
- All its `song_links`

No need to manually delete related rows.

### Code to write:
```js
app.delete('/releases/:releaseId', authMiddleware, async (req, res) => {
  try {
    const db = require('./db')
    const { releaseId } = req.params

    const result = await db.query(
      `DELETE FROM releases WHERE slug = $1 AND user_id = $2 RETURNING slug`,
      [releaseId, req.user.id]
    )

    if (result.rows.length === 0)
      return res.status(404).json({ success: false, error: 'Release not found' })

    res.json({ success: true, message: `Release ${releaseId} deleted` })
  } catch (error) {
    res.status(500).json({ success: false, error: error.message })
  }
})
```
